## 第一章：认识 Neko Shell

### 1.1 什么是 Neko Shell？ 🤔

**Neko Shell** 是一款专为运维工程师、开发人员和系统管理员设计的远程服务器管理工具。它的名字来源于日语中的"猫"（Neko），寓意着工具像猫咪一样灵活、敏捷。

简单来说，Neko Shell 可以帮助你：

- 🔐 **远程登录** Linux 服务器，执行命令
- 📁 **传输文件**，上传下载服务器上的文件
- 📊 **监控服务器**状态，查看 CPU、内存、磁盘、网络使用情况
- 🐳 **管理容器**，操作 Docker 容器、镜像和 Compose 项目
- 🚇 **搭建隧道**，实现端口转发和内网穿透
- 🖥️ **远程桌面**，通过 VNC 协议远程控制服务器
- 📋 **沉淀常用操作**，通过快捷命令、任务预设和工作区模板提升效率

### 1.2 为什么选择 Neko Shell？ ✨

市面上有很多远程连接工具，比如 Xshell、MobaXterm、Termius 等，那为什么还要选择 Neko Shell 呢？

| 特点 | 说明 |
|------|------|
| 🆓 **完全免费开源** | 基于 GPL v3 协议，无需付费，无功能限制 |
| 🎨 **现代化界面** | 类似 IDE 的界面设计，支持深色、浅色、护眼和跟随系统 |
| 🚀 **预览版可直接运行** | 0.1 预览版优先提供 Linux 二进制包，解压即可启动 |
| 🔧 **功能聚焦个人工作流** | SSH、SFTP、VNC、隧道、监控、Docker 等常用能力一体化 |
| 🌍 **动态语言** | 当前预览版默认提供简体中文界面，设置页只显示运行时真实可用的语言 |
| 🔒 **安全可靠** | 支持密钥认证，配置文件加密存储 |
| 🧰 **便于沉淀经验** | 支持快捷命令、命令宏、任务预设和工作区模板 |

### 1.3 适用人群 👥

- 🧑‍💻 **运维工程师**：日常服务器维护、故障排查、批量巡检
- 👨‍💻 **后端开发者**：部署应用、查看日志、远程调试
- 🎓 **学生和学习者**：学习 Linux、练习命令
- 🏢 **系统管理员**：批量管理多台服务器
- 🔧 **嵌入式开发者**：串口调试、设备管理

## 第二章：安装与启动

### 2.1 系统要求 📋

在安装 Neko Shell 之前，请确保你的电脑满足以下要求：

| 操作系统 | 最低要求 |
|----------|----------|
| 🐧 Linux | 任意现代发行版（已优先按 Linux 预览版验证） |
| 🪟 Windows | 当前无正式预览包，后续再补充发行验证 |
| 🍎 macOS | 当前无正式预览包，后续再补充发行验证 |

### 2.2 下载安装 📥

#### Linux 预览版（推荐）

```bash
# 1. 下载并解压预览包
tar -xzf neko-shell-0.1.0-preview-linux-<arch>-binary.tar.gz
cd neko-shell-0.1.0-preview-linux-<arch>-binary

# 2. 直接启动
./neko-shell/neko-shell
```

#### Linux 从源码运行

```bash
# 克隆仓库
git clone https://github.com/neko-shell/Neko_Shell.git
cd Neko_Shell

# 安装依赖
pip install -e ".[crypto,vnc]"

# 运行
python -m neko_shell
```

#### Windows / macOS 状态

当前 `0.1` 预览版不提供 Windows 和 macOS 的正式安装包。本手册后续若涉及这两个平台的描述，以“规划中”理解即可，不作为本次预览版承诺。

### 2.3 命令行参数 ⌨️

Neko Shell 支持丰富的命令行参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `--theme THEME` | 指定主题 (`dark`/`light`/`eye_care`/`auto`) | `neko-shell --theme eye_care` |
| `--config-dir DIR` | 指定配置目录 | `neko-shell --config-dir /path/to/config` |
| `--debug` | 启用调试模式 | `neko-shell --debug` |
| `--encrypt-config` | 加密配置文件 | `neko-shell --encrypt-config` |
| `--decrypt-config` | 解密配置文件 | `neko-shell --decrypt-config` |
| `--runtime-summary` | 输出运行摘要后退出 | `neko-shell --runtime-summary` |
| `--self-check` | 执行预览版自检并输出诊断结果 | `neko-shell --self-check` |
| `--acceptance-checklist` | 输出 0.1 预览版验收清单 | `neko-shell --acceptance-checklist` |
| `--export-acceptance-checklist FILE` | 导出 0.1 预览版验收清单 | `neko-shell --export-acceptance-checklist ./neko-shell-acceptance.md` |
| `--issue-template` | 输出可直接填写的问题反馈模板 | `neko-shell --issue-template` |
| `--export-issue-template FILE` | 导出问题反馈模板到指定文件 | `neko-shell --export-issue-template ./neko-shell-issue.md` |
| `--export-support-bundle FILE` | 导出预览版支持包到指定文件 | `neko-shell --export-support-bundle ./neko-shell-support-bundle.zip` |
| `--export-diagnostic FILE` | 导出诊断报告到指定文件 | `neko-shell --export-diagnostic ./neko-shell-diagnostics.txt` |
| `--smoke-test` | 执行一次 GUI 启动验证后退出 | `neko-shell --smoke-test` |
| `--version` | 显示版本信息 | `neko-shell --version` |

### 2.4 首次启动 👋

首次启动 Neko Shell 时，你会看到以下界面：

1. **欢迎界面**：显示 Neko Shell 的 Logo 和版本信息
2. **设备列表**：左侧空白区域，等待你添加服务器
3. **主工作区**：右侧空白区域，连接服务器后显示终端或文件浏览器

> 💡 **小贴士**：首次使用建议先阅读本手册，了解各项功能。

## 第四章：设备管理

### 4.1 添加第一个服务器连接 🎯

现在让我们添加你的第一个服务器连接！

#### 步骤一：打开新增配置窗口

有三种方式可以打开：

1. 点击菜单 **📁 文件 → ➕ 新增配置**
2. 点击工具栏的 **➕** 按钮
3. 在设备列表空白处 **右键 → 新增配置**
4. 使用快捷键 **`Ctrl+N`**

#### 步骤二：选择连接类型

在弹出的窗口中，首先选择连接类型：

| 类型 | 图标 | 用途 |
|------|------|------|
| **SSH** | 🔐 | 远程登录服务器，执行命令（最常用） |
| **SFTP** | 📁 | 安全文件传输，上传下载文件 |
| **FTP/FTPS** | 📤 | 传统文件传输协议 |
| **Serial** | 🔧 | 串口连接（嵌入式开发用） |
| **TCP** | 🌐 | TCP 网络调试 |
| **UDP** | 📡 | UDP 网络调试 |
| **VNC** | 🖥️ | 远程桌面 |

> 💡 **小贴士**：大多数情况下，你应该选择 **SSH** 类型，因为它同时支持终端和文件传输。

#### 步骤三：填写连接信息

**SSH 连接配置详解**：

```
┌─────────────────────────────────────────────────────────┐
│  ➕ 新增 SSH 连接                                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  📝 基本信息                                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 名称：     [生产服务器-Web-01          ]        │   │
│  │ 分组：     [生产环境                    ] ▼     │   │
│  │ 标签：     [web, nginx, 生产            ]        │   │
│  │ 描述：     [主站 Web 服务器              ]        │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  🌐 连接信息                                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 主机地址： [192.168.1.100             ]          │   │
│  │ 端口：     [22                         ]          │   │
│  │ 跳板机：   [                          ] (可选)   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  🔐 认证方式                                            │
│  ○ 密码认证  ● 密钥认证                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 用户名：   [root                       ]          │   │
│  │ 密钥文件： [~/.ssh/id_rsa        ] [浏览]       │   │
│  │ 密钥密码： [                          ] (可选)   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  ⚙️ 高级设置                                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 连接超时： [30           ] 秒                     │   │
│  │ 保活间隔： [60           ] 秒                     │   │
│  │ 自动重连： [☑] 启用                             │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│                          [取消]  [测试连接]  [保存]    │
└─────────────────────────────────────────────────────────┘
```

**各字段说明**：

| 字段 | 是否必填 | 说明 | 示例 |
|------|----------|------|------|
| 名称 | ✅ 必填 | 给这个连接起一个容易识别的名字 | 生产服务器-01 |
| 分组 | ⚪ 选填 | 把服务器分类管理 | 生产环境 / 测试环境 |
| 标签 | ⚪ 选填 | 添加标签便于搜索和筛选 | web, nginx |
| 描述 | ⚪ 选填 | 备注信息 | 主站 Web 服务器 |
| 主机地址 | ✅ 必填 | 服务器的 IP 地址或域名 | 192.168.1.100 或 server.example.com |
| 端口 | ✅ 必填 | SSH 服务端口 | 默认 22 |
| 跳板机 | ⚪ 选填 | 通过跳板机连接 | 跳板机配置名 |
| 用户名 | ✅ 必填 | 登录用户名 | root 或 ubuntu |
| 密码/密钥 | ✅ 必填 | 认证方式（二选一） | 见下方说明 |

#### 步骤四：选择认证方式

**方式一：密码认证** 🔑

最简单的方式，只需要输入：
- 用户名
- 密码

> ⚠️ **注意**：密码会加密保存在本地配置文件中，但建议使用密钥认证更安全。

**方式二：密钥认证** 🔐

更安全的方式，需要：
- 用户名
- 私钥文件路径（如 `~/.ssh/id_rsa`）
- 密钥密码（如果私钥有密码的话）

**如何生成 SSH 密钥？**

```bash
# 在本地电脑执行以下命令
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 按提示操作，可以选择：
# - 保存位置（默认 ~/.ssh/id_rsa）
# - 密钥密码（可选）

# 生成后，将公钥上传到服务器
ssh-copy-id -i ~/.ssh/id_rsa.pub user@server-ip
```

**方式三：SSH Agent** 🤖

如果你已经将密钥添加到 SSH Agent，可以选择使用 SSH Agent 认证，无需再指定密钥文件。

#### 步骤五：高级设置

| 设置 | 说明 | 默认值 |
|------|------|--------|
| 连接超时 | 建立连接的最长等待时间 | 30 秒 |
| 保活间隔 | 发送心跳包保持连接 | 60 秒 |
| 自动重连 | 断线后自动重新连接 | 启用 |
| 编码 | 终端字符编码 | UTF-8 |

#### 步骤六：测试并保存

1. 点击 **[测试连接]** 按钮，验证配置是否正确
2. 如果显示 ✅ 连接成功，点击 **[保存]**
3. 新添加的服务器会出现在左侧设备列表中

### 4.2 管理服务器分组 📂

当你的服务器越来越多时，分组管理可以帮助你更好地组织它们。

#### 创建分组

分组在添加/编辑配置时设置：

1. 编辑或新增配置
2. 在"分组"字段输入新的分组名称
3. 保存后，该分组自动创建

#### 分组操作

```

│   ├── 🟢 Web-01 (192.168.1.100)

│   └── ⚪ Cache-01 (192.168.1.102)
├── 📁 测试环境
│   ├── ⚪ Test-Web-01 (192.168.2.100)
│   └── ⚪ Test-DB-01 (192.168.2.101)
├── 📁 收藏
│   └── 🟢 Web-01 (192.168.1.100)
└── 📁 最近连接
```

### 5.1 连接到服务器 🔗

添加完服务器配置后，连接服务器非常简单：

**方式一：双击连接**

**方式二：右键连接**
- 右键点击服务器 → **连接**

## 第六章：SFTP 文件管理

### 6.1 什么是 SFTP？ 🤔

SFTP（SSH File Transfer Protocol）是一种安全的文件传输协议，它使用 SSH 加密通道传输文件，比传统的 FTP 更安全。

### 6.2 打开文件浏览器 📂

#### 方式一：直接连接 SFTP / FTP

如果添加的是 SFTP 或 FTP 类型配置，双击连接会直接打开文件浏览器。

#### 方式二：从 SSH 连接打开

1. 在左侧连接树中选中 SSH 连接
2. 使用任意一种入口：
   - 右键 → **打开文件浏览器**
   - 在 `Ctrl+P` 快速打开里选择 **文件 · 连接名**
   - 在 **文件 → 收藏连接 / 最近连接** 中进入对应连接子菜单
   - 在已打开的终端里使用右键菜单
3. 文件浏览器会在右侧工作区打开，并保留当前终端标签

```

│ 📁 文件浏览器 - Web-01                          [🔄] [🏠] [⬆️] │

├──────────────────────────────────────────────────────────────────┤

│  📁 .cache                                                       │
│  📁 .config                                                      │

│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 6.3 浏览目录 📁

#### 进入目录

- 🖱️ **双击** 目录名进入该目录
- ⌨️ 选中目录后按 **`Enter`**

#### 返回上级目录

- 点击工具栏 **⬆️** 按钮
- 或双击第一行的 **📁 ..**

#### 刷新目录

- 点击工具栏 **🔄** 按钮
- 或按快捷键 **`F5`**

#### 跳转到指定路径

1. 点击路径输入框
2. 输入目标路径（如 `/var/log`）
3. 按 **`Enter`** 跳转

#### 按名称筛选和排序

- 使用顶部 **筛选** 输入框按名称过滤文件和目录
- 使用 **排序** 与 **升序/降序** 组合快速切换浏览方式
- 需要分享路径时，可使用工具栏或右键菜单里的 **复制当前路径 / 复制路径**

### 6.4 文件上传和下载 📤📥

#### 上传文件（从本地到服务器）

**方式一：拖拽上传**

1. 在本地电脑上选中要上传的文件或文件夹
2. 🖱️ 拖拽到文件浏览器窗口中
3. 等待上传完成

**方式二：右键菜单上传**

1. 在文件浏览器空白处 **右键**
2. 选择 **上传** 或 **上传目录**
3. 在弹出的对话框中选择本地文件
4. 点击 **打开** 开始上传

上传后，任务会进入右侧 **传输队列**，可以查看进度、失败项和批次摘要。

#### 下载文件（从服务器到本地）

**方式一：右键下载**

1. 选中要下载的文件或文件夹
2. **右键 → 📥 下载**
3. 选择本地保存位置
4. 等待下载完成

下载任务同样会进入右侧 **传输队列**，可继续查看进度、失败项、重试和批次历史。

#### 断点续传

如果上传/下载过程中断：

1. 重新开始传输
2. Neko Shell 会检测已存在的部分文件
3. 询问是否从断点继续传输

#### 批量上传下载

选择多个文件的方式：
- 按住 **`Ctrl`** 键点击多个文件
- 按住 **`Shift`** 键选择连续的文件
- 按 **`Ctrl+A`** 全选当前目录

然后进行上传或下载操作。

3. 在确认对话框中点击 **确定**

### 6.6 压缩和解压 📦

#### 压缩文件

1. 选中要压缩的文件或文件夹
2. **右键 → 📦 压缩**
3. 选择压缩格式：
   - `.zip`
   - `.tar.gz`
   - `.tar.bz2`
4. 输入压缩包名称
5. 点击 **确定**

#### 解压文件

1. 选中压缩包
2. **右键 → 📦 解压**
3. 选择解压位置
4. 点击 **确定**

支持的格式：

   - 数字模式：如 `755`
   - 或勾选读/写/执行权限

#### 修改所有者 (chown)

1. 右键文件或文件夹
2. 选择 **👤 所有者**
3. 输入用户名和/或组名
4. 点击 **确定**

### 6.8 内置文本编辑器 ✏️

Neko Shell 内置了一个代码编辑器，可以直接编辑服务器上的文件。

#### 打开编辑器

- 或 **右键 → ✏️ 编辑**

编辑器支持：
- 🎨 语法高亮（支持多种编程语言）
- 📝 行号显示
- 🔍 查找和替换
- 🔤 编码选择

#### 保存文件

- 点击 **保存** 按钮或按 **`Ctrl+S`**
- 文件会自动上传到服务器

#### 编码问题

如果打开文件显示乱码：

1. 点击编辑器底部的编码选择器
2. 选择正确的编码（如 `UTF-8`、`GBK` 等）
3. 文件会以新编码重新打开

## 第八章：SSH 隧道配置

### 8.1 什么是 SSH 隧道？ 🚇

SSH 隧道（也叫端口转发）是一种通过 SSH 连接建立加密通道的技术，可以：

- 🔐 安全访问远程服务（如数据库）
- 🌐 绕过防火墙限制
- 🔗 连接内网服务

Neko Shell 支持三种隧道类型：
- **本地端口转发** — 将远程服务映射到本地
- **远程端口转发** — 将本地服务暴露到远程
- **动态端口转发** — 创建 SOCKS 代理

### 8.2 打开隧道管理 🚇

点击菜单 **🔧 工具 → 🚇 SSH 隧道**，打开隧道管理窗口：

```
┌─────────────────────────────────────────────────────────────────┐
│  🚇 SSH 隧道管理                                                 │
├─────────────────────────────────────────────────────────────────┤
│  名称          类型      本地端口    远程端口    状态    操作   │
├─────────────────────────────────────────────────────────────────┤
│  MySQL隧道     本地      3306       3306       🟢运行   [停][删]│
│  Redis隧道     本地      6379       6379       ⚪停止   [启][删]│
│  SOCKS代理     动态      1080       -          ⚪停止   [启][删]│
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                        [➕ 新增隧道]             │
└─────────────────────────────────────────────────────────────────┘
```

### 8.3 本地端口转发 📍

本地端口转发将远程服务映射到本地端口，让你可以像访问本地服务一样访问远程服务。

#### 场景示例

假设：
- 远程服务器上运行着 MySQL（端口 3306）
- 但 MySQL 只监听 localhost，不允许外部直接连接
- 你想用本地电脑上的 MySQL 客户端连接

#### 配置步骤

1. 点击 **➕ 新增隧道**
2. 选择连接的服务器
3. 填写配置：

| 字段 | 值 | 说明 |
|------|-----|------|
| 名称 | MySQL隧道 | 给隧道起个名字 |
| 类型 | 本地端口转发 | 选择转发类型 |
| 本地地址 | 127.0.0.1 | 本地监听地址 |
| 本地端口 | 3306 | 本地端口（可与远程不同） |
| 远程地址 | 127.0.0.1 | 远程服务地址 |
| 远程端口 | 3306 | 远程服务端口 |

4. 点击 **保存**
5. 点击 **启动** 按钮

#### 使用隧道

隧道启动后，在本地电脑上：

```bash
# 使用 MySQL 客户端连接
mysql -h 127.0.0.1 -P 3306 -u root -p
```

就好像 MySQL 运行在你本地一样！

### 8.4 远程端口转发 🌐

远程端口转发将本地服务暴露到远程服务器上，让其他人可以通过远程服务器访问你的本地服务。

#### 场景示例

假设：
- 你本地运行着一个 Web 服务（端口 8080）
- 你想让同事通过公网服务器访问你的本地服务

#### 配置步骤

| 字段 | 值 | 说明 |
|------|-----|------|
| 名称 | Web服务转发 | 给隧道起个名字 |
| 类型 | 远程端口转发 | 选择转发类型 |
| 远程地址 | 0.0.0.0 | 远程服务器监听地址 |
| 远程端口 | 8888 | 远程服务器端口 |
| 本地地址 | 127.0.0.1 | 本地服务地址 |
| 本地端口 | 8080 | 本地服务端口 |

启动后，其他人可以通过 `http://服务器IP:8888` 访问你本地的服务。

### 8.5 动态端口转发 (SOCKS) 🔄

动态端口转发创建一个 SOCKS 代理，让你可以通过远程服务器访问网络。

#### 配置步骤

| 字段 | 值 | 说明 |
|------|-----|------|
| 名称 | SOCKS代理 | 给隧道起个名字 |
| 类型 | 动态端口转发 | 选择转发类型 |
| 本地地址 | 127.0.0.1 | 本地监听地址 |
| 本地端口 | 1080 | 本地代理端口 |

#### 使用 SOCKS 代理

启动后，配置浏览器或应用程序使用 SOCKS5 代理：
- 代理地址：127.0.0.1
- 代理端口：1080

| 操作 | 说明 |
|------|------|
| 启动 | 启动已停止的隧道 |
| 停止 | 停止运行中的隧道 |
| 编辑 | 修改隧道配置 |
| 删除 | 删除隧道配置 |

## 第九章：内网穿透 (FRP)

### 9.1 什么是内网穿透？ 🌍

内网穿透（NAT Traversal）是一种让内网服务可以被外网访问的技术。当你没有公网 IP 时，可以通过 FRP 将内网服务暴露到公网。

#### 典型场景

- 🏠 在家开发，想让客户预览你的网站
- 🏢 公司内网服务器，需要从外部访问
- 📱 开发移动应用，需要回调内网的接口
- 🎮 搭建游戏服务器，让朋友加入

### 9.2 准备工作 📋

使用 FRP 需要：

1. **一台有公网 IP 的服务器** — 作为 FRP 服务端
2. **Neko Shell** — 内置 FRP 客户端功能

#### 搭建 FRP 服务端

在你的公网服务器上：

```bash
# 下载 FRP（Neko Shell 使用 v0.61.1）
wget https://github.com/fatedier/frp/releases/download/v0.61.1/frp_0.61.1_linux_amd64.tar.gz
tar -xzf frp_0.61.1_linux_amd64.tar.gz
cd frp_0.61.1_linux_amd64

# 编辑服务端配置
cat > frps.toml << EOF
bindPort = 7000
auth.token = "your-secret-token"
EOF

# 启动服务端
./frps -c frps.toml
```

> 💡 **提示**：Neko Shell 可以自动下载并部署 FRP 服务端，见 9.4 节。

### 9.3 配置 FRP 客户端 🔧

在 Neko Shell 中：

1. 点击菜单 **🔧 工具 → 🌐 内网穿透**
2. 配置 FRP 服务器信息：

```
┌─────────────────────────────────────────────────────────────────┐
│  🌐 内网穿透 (FRP) 配置                                          │
├─────────────────────────────────────────────────────────────────┤
│  FRP 服务器设置                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 服务器地址：[123.45.67.89         ]                      │   │
│  │ 服务器端口：[7000                ]                       │   │
│  │ 认证令牌：  [your-secret-token    ]                      │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  穿透规则列表                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 名称    类型    本地端口    远程端口    状态              │   │
│  │ Web服务 HTTP   8080       8080       ⚪停止            │   │
│  │ SSH穿透 TCP    22         6000       ⚪停止            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│                           [➕ 添加规则]  [💾 保存配置]           │
└─────────────────────────────────────────────────────────────────┘
```

### 9.4 自动部署 FRP 服务端 🚀

Neko Shell 支持自动在服务器上部署 FRP：

1. 打开 FRP 配置窗口
2. 点击 **自动部署服务端**
3. 选择一台已配置的服务器
4. 点击 **部署**
5. Neko Shell 会自动：
   - 下载 FRP
   - 上传到服务器
   - 生成配置文件
   - 启动服务端

### 9.5 添加穿透规则 ➕

#### TCP 穿透

适用于：SSH、数据库、游戏服务器等

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 名称 | SSH穿透 | 规则名称 |
| 类型 | TCP | 穿透类型 |
| 本地地址 | 127.0.0.1 | 本地服务地址 |
| 本地端口 | 22 | 本地服务端口 |
| 远程端口 | 6000 | FRP 服务器开放的端口 |

启动后，可以通过 `ssh -p 6000 user@frp-server-ip` 连接。

#### HTTP 穿透

适用于：网站、Web 应用、API 等

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 名称 | 我的网站 | 规则名称 |
| 类型 | HTTP | 穿透类型 |
| 本地地址 | 127.0.0.1 | 本地服务地址 |
| 本地端口 | 8080 | 本地服务端口 |
| 自定义域名 | myapp.example.com | 可选，绑定域名 |

启动后，可以通过 `http://frp-server-ip:8080` 或绑定的域名访问。

| 设置 | 说明 |
|------|------|
| 界面主题 | 切换深色、浅色、护眼或跟随系统 |
| 语言 | 仅显示当前运行时真实可用的界面语言 |

| 终端字号 | 设置字体大小 |
| 终端配色 | 选择预设配色方案 |

### 13.3 连接设置 🔗

| 设置 | 说明 | 默认值 |
|------|------|--------|
| 连接超时 | 建立连接的超时时间（秒） | 30 |
| 保活间隔 | SSH 心跳间隔（秒） | 60 |
| 重连次数 | 断线自动重连次数 | 3 |
| 重连间隔 | 重连间隔时间（秒） | 5 |
| 默认编码 | 终端字符编码 | UTF-8 |

### 13.4 安全设置 🔐

| 设置 | 说明 |
|------|------|
| 主密码 | 设置主密码后，所有配置加密存储 |
| 自动锁定 | 空闲多少分钟后自动锁定 |
| 锁定界面 | 锁定后需要输入主密码解锁 |

> 💡 **建议**：如果你在公共电脑上使用，强烈建议设置主密码。

### 13.5 配置文件位置 📁

Neko Shell 的配置文件存储在以下位置：

| 操作系统 | 路径 |
|----------|------|
| 🪟 Windows | `%APPDATA%\NekoShell\` |
| 🍎 macOS | `~/Library/Application Support/NekoShell/` |
| 🐧 Linux | `~/.config/NekoShell/` |

主要文件：
- `config.json` — 应用配置
- `connections.json` — 连接配置
- `settings.json` — 用户设置

## 第十五章：常见问题解答

### Q1: 连接服务器时提示"连接超时" ⏱️

**可能原因及解决方案**：

1. **IP 地址或端口错误**
   - 检查配置中的主机地址和端口是否正确
   - 确认 SSH 服务端口（默认 22）

2. **网络不通**
   ```bash
   # 测试网络连通性
   ping 服务器IP
   
   # 测试端口是否开放
   telnet 服务器IP 端口
   # 或
   nc -zv 服务器IP 端口
   ```

3. **服务器防火墙阻止**
   - 检查服务器防火墙规则
   - 确认 SSH 端口已开放

4. **增加超时时间**
   - 在设置中将连接超时时间调大

### Q2: SSH 密钥认证失败 🔑

**检查清单**：

1. **私钥文件格式**
   - OpenSSH 格式：以 `-----BEGIN OPENSSH PRIVATE KEY-----` 开头
   - PuTTY 格式：以 `PuTTY-User-Key-File-...` 开头

2. **私钥权限**（Linux/macOS）
   ```bash
   chmod 600 ~/.ssh/id_rsa
   ```

3. **公钥是否已添加**
   ```bash
   # 在服务器上检查
   cat ~/.ssh/authorized_keys
   ```

4. **SELinux 问题**（CentOS/RHEL）
   ```bash
   restorecon -R -v ~/.ssh
   ```

### Q3: 终端显示乱码 🔤

**解决方案**：

1. 检查服务器编码
   ```bash
   echo $LANG
   # 应显示类似 en_US.UTF-8 或 zh_CN.UTF-8
   ```

2. 设置服务器编码
   ```bash
   export LANG=en_US.UTF-8
   ```

3. 在 Neko Shell 设置中调整终端编码

### Q4: 如何传输大文件？📦

**建议**：

1. 使用 SFTP 直接拖拽传输（支持断点续传）
2. 对于超大文件，建议在终端使用 rsync：
   ```bash
   rsync -avz --progress 本地文件 user@server:远程路径
   ```

### Q5: 忘记主密码怎么办？🔐

**不幸的是**，主密码用于加密配置，一旦忘记无法恢复。

解决方案：
1. 删除配置文件目录（见 13.5 节）
2. 重新导入之前导出的配置备份

> 💡 **建议**：定期导出配置备份，并妥善保管主密码。

### Q6: 如何同时操作多台服务器？🔄

**方法**：

1. 使用 **同步输入** 功能
   - 打开多个终端标签
   - 启用同步输入
   - 在任一终端输入命令，所有终端同步执行

2. 使用任务预设功能批量执行

### Q7: Windows 上无法使用快捷键？🪟

Windows 用户注意：
- 复制：`Ctrl+Shift+C` 或 `Ctrl+Insert`
- 粘贴：`Ctrl+Shift+V` 或 `Shift+Insert`
- 也可以在设置中自定义快捷键

### Q8: 如何报告问题？🐛

推荐按下面顺序操作：

1. 在 GUI 中打开 **帮助 → 关于**，优先点击 **导出支持包**
2. 如需命令行方式，也可以执行：
   ```bash
   neko-shell --export-support-bundle ./neko-shell-support-bundle.zip
   neko-shell --self-check
   neko-shell --acceptance-checklist
   neko-shell --export-acceptance-checklist ./neko-shell-acceptance.md
   neko-shell --issue-template
   neko-shell --export-issue-template ./neko-shell-issue.md
   neko-shell --export-diagnostic ./neko-shell-diagnostics.txt
   ```
3. 支持包默认包含运行摘要、运行诊断、验收清单和反馈模板；日志与配置内容涉及隐私，建议筛选后再单独补充
4. 如有必要，再打开 **配置目录** 和 **日志目录**，补充关键文件内容
5. 访问 [GitHub Issues](https://github.com/neko-shell/Neko_Shell/issues)
6. 点击 **New Issue**
7. 描述问题：
   - 操作系统和版本
   - Neko Shell 版本
   - 问题复现步骤
   - 期望结果和实际结果
   - 诊断报告、截图或关键日志
